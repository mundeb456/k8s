# Kubernetes Notes: Deployment, StatefulSet, Service & Headless Service

---

# 1. Deployment

## What is a Deployment?

A Deployment is used to manage **stateless applications**.

Examples:
- Nginx
- React applications
- REST APIs
- Spring Boot
- Node.js applications

Pods created by a Deployment are interchangeable.

---

## Deployment YAML

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3

  selector:
    matchLabels:
      app: my-app

  template:
    metadata:
      labels:
        app: my-app

    spec:
      containers:
      - name: nginx
        image: nginx
        ports:
        - containerPort: 80
```

---

## Background Working

```
kubectl apply

      │
      ▼

API Server

      │
      ▼

etcd

      │
      ▼

Deployment Controller

      │
      ▼

ReplicaSet

      │
      ▼

Pods
```

Deployment Controller ensures desired replicas are always running.

# 2. StatefulSet

## What is StatefulSet?

StatefulSet manages applications requiring

- Stable Pod names
- Stable network identity
- Persistent storage
- Ordered deployment

Examples

- MySQL
- PostgreSQL
- MongoDB
- Kafka
- Cassandra
- ZooKeeper

---

## StatefulSet YAML

```yaml
apiVersion: apps/v1
kind: StatefulSet

metadata:
  name: mysql

spec:
  serviceName: mysql

  replicas: 3

  selector:
    matchLabels:
      app: mysql

  template:
    metadata:
      labels:
        app: mysql

    spec:
      containers:
      - name: mysql
        image: mysql:8
        ports:
        - containerPort: 3306

  volumeClaimTemplates:
  - metadata:
      name: data
    spec:
      accessModes:
      - ReadWriteOnce

      resources:
        requests:
          storage: 1Gi
```

---

## Storage

Each Pod gets its own PVC.

```
mysql-0
    │
PVC-0
    │
PV-0

mysql-1
    │
PVC-1
    │
PV-1

mysql-2
    │
PVC-2
    │
PV-2
```

If mysql-1 crashes

```
mysql-1 ❌

↓

mysql-1 ✅
```

Same Pod name

Same PVC

Same data

# Deployment vs StatefulSet

| Feature | Deployment | StatefulSet |
|----------|------------|-------------|
| Application | Stateless | Stateful |
| Pod Name | Random | Fixed |
| Storage | Shared/Ephemeral | Dedicated PVC |
| Network Identity | Dynamic | Stable |
| Scaling | Any order | Sequential |
| Update | Rolling | Ordered |
| Example | Nginx | MySQL |

# 3. Kubernetes Service

A Service exposes Pods using a stable IP and DNS.

```
Client

    │

Service

    │

Pods
```

---

## ClusterIP Service

```yaml
apiVersion: v1
kind: Service

metadata:
  name: my-service

spec:
  selector:
    app: my-app

  ports:
  - port: 80
    targetPort: 8080

  type: ClusterIP
```

Flow

```
Client

↓

Service

↓

Pod-1

Pod-2

Pod-3
```

---

## NodePort

```yaml
type: NodePort
```

Accessible via

```
NodeIP:30080
```

---

## LoadBalancer

```yaml
type: LoadBalancer
```

Creates external Load Balancer (Cloud).

# 4. Headless Service

A Headless Service has

```yaml
clusterIP: None
```

Example

```yaml
apiVersion: v1
kind: Service

metadata:
  name: mysql

spec:
  clusterIP: None

  selector:
    app: mysql

  ports:
  - port: 3306
```

Instead of one Cluster IP, DNS returns individual Pod IPs.

```
mysql-0.mysql.default.svc.cluster.local

mysql-1.mysql.default.svc.cluster.local

mysql-2.mysql.default.svc.cluster.local
```

Used mainly with StatefulSets.

---

# 5. How Service Selects Pods

Service never selects Deployment.

Service never selects StatefulSet.

Service selects Pods.

Deployment

```
spec:

 selector:
    matchLabels:
      app: my-app
```

Pod

```
template:

 metadata:

  labels:

    app: my-app
```

Service

```yaml
spec:

 selector:

   app: my-app
```

Flow

```
Deployment

        │

Creates

        │

Pods

labels:

app=my-app

        ▲

        │

Service selector

app=my-app
```

The selector matches **Pod labels**, not Deployment or StatefulSet metadata.

                Deployment

# Interview Questions

### Deployment
- Used for stateless applications.
- Pods are interchangeable.
- Uses ReplicaSet.

### StatefulSet
- Used for databases.
- Stable Pod names.
- Stable DNS.
- One PVC per Pod.
- Ordered deployment.

### Service
- Stable networking.
- Selects Pods using labels.
- Load balances traffic.

### Headless Service
- `clusterIP: None`
- No load balancing.
- Returns Pod IPs.
- Used with StatefulSets.

---

# Key Points to Remember

- Deployment → Stateless applications
- StatefulSet → Stateful applications
- Service → Stable networking
- Headless Service → Stable Pod DNS
- Service selects Pods using labels
- StatefulSet gives each Pod a unique identity
- Deployment Pods are replaceable
